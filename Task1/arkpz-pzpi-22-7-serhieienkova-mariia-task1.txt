1. BUSINESS REQUIREMENTS
1.1 Background (Передумови)


Сучасний світ стикається з демографічними змінами, які ведуть до збільшення частки літніх людей у суспільстві. Ця категорія населення потребує спеціального догляду, постійного моніторингу здоров’я та вчасного реагування на зміни у стані організму. Сім'ї, які піклуються про своїх літніх родичів, нерідко відчувають труднощі через відсутність ефективних інструментів для дистанційного спостереження за здоров’ям. У свою чергу, медичні працівники потребують засобів, які спрощують обробку даних і забезпечують оперативну реакцію на критичні зміни.
Програмна система для догляду за літніми людьми створюється для вирішення цієї проблеми. Вона дозволяє дистанційно відслідковувати ключові життєві показники (пульс, температуру тіла, артеріальний тиск), автоматично інформувати родичів або лікарів у разі критичних змін, а також надавати рекомендації щодо подальших дій. 
Основна мета розробки — забезпечити безпеку та комфорт літніх людей, знизити емоційне та фізичне навантаження на родичів і доглядальників, а також покращити ефективність медичного нагляду.






1.2	Business Opprtunity (Бізнес можливості)


Ринок технологій для догляду за літніми людьми стрімко розвивається. Існує чимало продуктів для моніторингу здоров’я, але більшість з них мають обмежений функціонал, відсутність підтримки багатьох мов або незручний інтерфейс. Пропонована система вирізняється інтеграцією IoT-пристроїв, аналізом даних з допомогою аналітичних алгоритмів та підтримкою багатомовності. 
Існуючі аналоги системи:
-	CarePredict – система для моніторингу активності літніх людей.
Вона використовує спеціальні датчики для відслідковування рухів і поведінки, але не забезпечує повного моніторингу життєвих показників (таких як пульс чи тиск).
-	GrandCare – платформа для дистанційного моніторингу, яка
дозволяє відслідковувати стан здоров’я літніх людей через IoT-пристрої. Водночас вона складна в налаштуванні та використанні.
     Модель монетизації передбачає включати:
-	Продаж IoT-пристроїв з базовим функціоналом.
-	Передплату на розширені функції аналізу даних та підтримку.
-	Інтеграцію з медичними організаціями для спрощення процесу обслуговування пацієнтів.
          Цільовий ринок — країни з розвиненою інфраструктурою медичних послуг і високим рівнем використання IoT. Прогнозується, що попит на систему зростатиме завдяки демографічним трендам і популярності рішень для здоров’я.


2	VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)
2.1	 Vision Statement (Окреслення концепції)


Система для догляду за літніми людьми спрямована на покращення якості життя цієї категорії населення шляхом впровадження сучасних технологій моніторингу здоров'я. Вона призначена для  дистанційного спостереження за життєвими показниками, автоматичного інформування про критичні ситуації та надання рекомендацій щодо дій. 
Метою системи є створення безпечного середовища для літніх людей, а також підтримка їх родичів і медичних працівників через інтуїтивно зрозумілий інтерфейс та інтеграцію з IoT-пристроями. 
Система стане платформою, яка забезпечить повний спектр функцій для моніторингу здоров’я літніх людей, включаючи збір даних у реальному часі, аналітику та автоматичні сповіщення. Це дозволить не лише зменшити ризики, пов’язані з раптовими змінами стану здоров’я, але й покращити комунікацію між пацієнтами, їхніми близькими та медичними працівниками. 


2.2	Major Features (Головна функціональність)


Система забезпечує наступну функціональність:
-	MF-1 Моніторинг життєвих показників здоров'я — система
дозволяє відслідковувати показники, такі як пульс, температура тіла, артеріальний тиск. Дані збираються автоматично за допомогою                                                інтегрованих IoT-пристроїв.

-	MF-2 Аналітика та рекомендації — система обробляє дані за
допомогою аналітичних алгоритмів для надання можливих ризиків.     Надання рекомендацій користувачам щодо покращення стану здоров'я, система може порекомендувати додаткові обстеження.
-	MF-3 Автоматичні сповіщення — у разі виявлення критичних
змін у стані здоров'я користувача (наприклад, підвищення артеріального тиску до небезпечного рівня або аномального серцевого ритму), система автоматично інформує медичних працівників, родичів або доглядальників. Це забезпечує своєчасну реакцію та допомогу у випадках надзвичайних ситуацій.
-	MF-4 Нагадування про прийом ліків і заплановані медичні
огляди, що допомагає уникнути пропусків. Така функція допомагає уникнути пропусків у лікуванні, що може бути критичним для людей похилого віку.
-	MF-5 Історія даних та звітність — система зберігає історію змін
життєвих показників для аналізу довгострокових тенденцій. Генерує звіти, які можуть бути використані для консультацій із медичними фахівцями. Це також дозволяє медичному персоналу краще оцінювати ефективність лікування.











3.	SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)
3.1	 Scope of Initial Release (Рамки первинного випуску)


Програмна система для догляду за літніми людьми передбачає створення комплексного рішення для дистанційного моніторингу життєвих показників таких, як пульс, температура тіла, артеріальний тиск, з використанням IoT-пристроїв. 
Метою першої версії системи є забезпечення базової функціональності для моніторингу стану здоров’я, сповіщення про аномалії та управління даними. Вона буде доступна через веб-платформу, з можливістю доступу для лікарів, родичів та доглядальників.
Основні функціональні можливості першого випуску:
1.	Back-end:
-	Реалізація RESTful API для взаємодії клієнтської частини з
     сервером і IoT-пристроями.
-	Аналіз показників: Алгоритми будуть здійснювати початковий
аналіз на предмет виявлення критичних змін у здоров'ї, таких як значні коливання пульсу або артеріального тиску.
-	Сповіщення: Система відправлятиме сповіщення до родичів,
лікарів або доглядальників при виявленні критичних змін у стані здоров’я користувача.
-	Забезпечення можливості кільком членам родини, лікарям та
доглядальникам мати доступ до профілю літньої людини, що дозволить зручніше контролювати стан її здоров’я.
-	Обробка запитів для управління користувачами, даними про 
     стан здоров'я та сповіщеннями.

-	Функціонал для реєстрації, авторизації та управління 
     профілями користувачів.
-	Автоматична обробка даних, отриманих з IoT-пристроїв, і
     формування рекомендацій або критичних сповіщень.
-	Імпорт та експорт даних для збереження історії медичних 
     показників.
-	Взаємодія з базою даних, сервер забезпечує обробку запитів на
     збереження, оновлення, видалення та отримання даних.
2.	IoT-пристрої:
-	Використання простих датчиків для вимірювання основних
показників здоров’я (пульс, температура тіла, артеріальний тиск) з можливістю підключення до серверу через Wi-Fi.
-	Передача зібраних даних на сервер для аналізу та обробки
     даних в реальному часі.
-	Автоматичне оновлення показників для відображення в
     інтерфейсі.
   Основна бізнес-логіка в першому випуску зосереджена на забезпеченні
базової роботи системи: збір даних з пристроїв, сповіщення у разі аномалій та підтримка багатомовного інтерфейсу. Адміністрування користувачів та моніторинг здоров’я зводяться до простих операцій для полегшення взаємодії з системою.





3.2	 Scope of Subsequent Releases (Рамки наступних випусків)


Наступні версії системи передбачають розширення функціональності, включаючи вдосконалені алгоритми для аналізу даних, додавання нових можливостей для адміністрування.
Перелік функцій для майбутніх випусків:
-	Покращений аналіз даних: Наступні версії включатимуть більш
     складні алгоритми для аналізу показників здоров’я, з можливістю прогнозування
     майбутніх аномалій та надання рекомендацій.
     щодо лікування.
-	Розширені функції адміністрування: Введення розширених прав
     доступу, можливості налаштування повідомлень, а також доступу до історії
     медичних даних на рівні лікарів і доглядальників.
-	Інтеграція з іншими типами пристроїв: Розширення підтримки для
      інших типів IoT-пристроїв (наприклад, пристрої для моніторингу рівня кисню в
      крові, електрокардіографи та інші медичні датчики).
-	Розширені функції адміністрування: Введення розширених прав
      доступу, можливості налаштування повідомлень, а також доступу до історії
      медичних даних на рівні лікарів і доглядальників.
-	Розроблена додаткова безпека та захист даних: Наступні версії
      системи будуть мати механізм шифрування даних та підтримку аутентифікації
      через двофакторну автентифікацію для підвищення рівня безпеки. 
-	Мобільна версія: Створення мобільних додатків для Android, що
      дозволить користувачам та родичам зручно отримувати доступ до даних і
      отримувати сповіщення на свої смартфони.



4.	BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)
4.3	 Operating Enviroment (Робоче середовище)


               Робоче середовище для розробки та функціонування програмної системи
       базується на сучасних технологіях і засобах, що забезпечують надійність,
       продуктивність та цілісність системи. У цьому розділі представлено мови
       програмування, фреймворки, бази даних та бібліотеки, які будуть використані під
       час реалізації проєкту.
 Мови програмування:
-	Go: використовується для розробки серверної частини. Підтримує
  роботу розробленого RESTful API та взаємодію із базою даних;
-	JavaScript: використовуватиметься для створення web-застосунку, що
  надаватиме користувацький інтерфейс для взаємодії із системою.
Інфраструктура серверної частини:
-	Gin: веб-фреймворк для створення RESTful API та організації серверної
       логіки;
-	PostgreSQL: СКБД для зберігання та взаємодією із базою даних
        користувачів, пацієнтів, отриманих показників тощо;
-	sqlx: бібліотека-розширення до стандартної бібліотеки database/sql,
        використовується для взамодії із базою даних;
-	JWT: система авторизації на основі токенів, що гарантує безпеку як
        даних, так і взаємодії із системою загалом.
Інфраструктура фронтенд частини:
-	React: фреймворк для створення веб-інтерфейсу;
-	Material-UI: бібліотека для полегшення та покращення стилізації веб
інтерфейсу;
-	Axious: браузерний http-клієнт для взаємодії із серверною частиною
сервісу.
